## 🌟 Features

-   ⚡️ **Lightning Fast Setup**: Get your API running in minutes
-   🔒 **Security First**: Pre-configured security with `helmet` and best practices
-   📦 **Modern Stack**: Express.js with TypeScript/JavaScript support

-   🛠️ **Developer Experience**: ESLint, Prettier, and Biome configurations

-   📊 **MongoDB Ready**: Pre-configured database setup
-   🧪 **Testing Ready**: Jest configuration included
-   📝 **API Documentation**: Swagger/OpenAPI setup included

## 🚀 Getting Started

### Quick Start

Choose your preferred version:

```bash
# Stable Version (Recommended for Production)
npx express-api-template

# Beta Version (Latest Features)
npx express-api-template@beta

# Canary Version (Experimental)
npx express-api-template@canary
```

### Installation Steps

1. **Update Dependencies (Optional)**

    ```bash
    npm update   # or yarn update / pnpm update
    ```

2. **Install Dependencies**

    ```bash
    npm install  # or yarn install / pnpm install
    ```

3. **Configure Environment**

    - Copy `.env.example` to `.env`
    - Update environment variables

    ```bash
    cp .env.example .env
    ```

4. **Start Development Server**
    ```bash
    npm run dev  # or yarn dev / pnpm dev
    ```

### 🔹 Available Templates

### 1. JavaScript MVC {#js-mvc}

> Traditional MVC pattern, perfect for quick starts and learning

**Best for:**

-   🎯 Quick prototypes
-   🎓 Learning Express.js
-   🚀 Small to medium projects
-   🔧 Simple CRUD applications
-   👨‍💻 Solo developers

**Key Features:**

-   Simple folder structure
-   Easy to understand
-   Quick to set up
-   Minimal boilerplate
-   Traditional MVC pattern

### 2. TypeScript MVC {#ts-mvc}

> Type-safe MVC architecture with enhanced developer experience

**Best for:**

-   🛡️ Type safety
-   🔍 Better IDE support
-   🐛 Reduced runtime errors
-   📝 Self-documenting code
-   🏗️ Medium-sized projects

**Key Features:**

-   Strong type checking
-   Enhanced IDE support
-   Better error catching
-   Interface definitions
-   Improved maintainability

### 3. JavaScript Modern {#js-modern}

> Feature-based architecture for scalable applications

**Best for:**

-   🏗️ Large applications
-   👥 Team collaboration
-   🔄 Microservices architecture
-   📦 Feature modularity
-   🔧 Complex business logic

**Key Features:**

-   Feature-based organization
-   Service layer abstraction
-   Better code isolation
-   Scalable architecture
-   Easy testing setup

### 4. TypeScript Modern {#ts-modern}

> Enterprise-grade architecture with full type safety

**Best for:**

-   🏢 Enterprise applications
-   🔐 Type-safe codebase
-   📈 Large-scale systems
-   👥 Large teams
-   🌐 Complex domains

**Key Features:**

-   Full type safety
-   Domain-driven design
-   Advanced patterns support
-   Strict boundaries
-   Enterprise-ready setup

### 🚀 Quick Template Selection

1. **Choose JavaScript MVC if:**

    - You're new to Express.js
    - Need a quick prototype
    - Simple project structure
    - Learning web development
    - Building a basic API

2. **Choose TypeScript MVC if:**

    - Want type safety
    - Familiar with MVC
    - Small to medium project
    - Need better IDE support
    - Value code documentation

3. **Choose JavaScript Modern if:**

    - Building larger applications
    - Need feature isolation
    - Multiple developers
    - Complex business logic
    - Microservices architecture

4. **Choose TypeScript Modern if:**
    - Enterprise application
    - Large development team
    - Need strict type safety
    - Complex domain logic
    - Long-term maintenance
